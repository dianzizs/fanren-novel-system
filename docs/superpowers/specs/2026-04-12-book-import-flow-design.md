# 书籍导入与详情页设计

## 概述

实现书籍导入后自动/半自动进入详情分析流程，包含进度展示和实时图谱更新。

## 用户流程

1. 用户在藏书页面点击「导入新书」上传 TXT 文件
2. 新书出现在列表中，状态为「待处理」
3. 用户点击「开始分析」按钮
4. 跳转到书籍详情页，开始后台索引处理
5. 详情页实时显示：
   - 处理进度（索引中...）
   - 人物卡片（随索引更新）
   - 知识图谱（随索引更新）
   - 事件时间线
6. 处理完成后，用户可与图谱交互、进行问答和续写

## API 设计

### 新增端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/books/{book_id}/status` | GET | 获取索引进度 |
| `/api/books/{book_id}/start-index` | POST | 开始后台索引 |

### 现有端点改动

- `POST /api/books` → 上传后自动设置 `status: "pending"`
- `GET /api/books/{book_id}/graph` → 返回图谱数据（当 status=ready）
- `DELETE /api/books/{book_id}` → 删除时清理所有关联数据

## 前端路由

### 路由设计
- `/` → 藏书列表页
- `/book/{book_id}` → 书籍详情页

### 单页应用结构
使用 URL hash 或查询参数实现路由跳转：
- `#/` 或 `?view=library`
- `#/book/fanren-1-500` 或 `?view=detail&book=fanren-1-500`

## 进度更新机制

### 轮询方案（推荐）
- 前端每 2-3 秒轮询 `/api/books/{book_id}/status`
- 进度更新时自动刷新图谱和相关数据

### 状态流转
```
pending → indexing → ready
              ↓
            error
```
